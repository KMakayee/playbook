# Checkpoint

Write a state snapshot to `tasks/checkpoint.md` with:

1. **Current phase** — Research / Plan / Implement / Other
2. **What's done** — Completed steps (brief)
3. **What's next** — Numbered steps with exact commands to run and file paths to read. A cold reader should be able to resume without asking questions.
4. **Artifacts in hand** — Trace files, temp outputs, partial results, uncommitted changes — anything produced this session that the next session needs to know about. Include paths.
5. **Key decisions made** — Choices that affect remaining work
6. **Open risks** — Anything unresolved

Keep it under 100 lines. Overwrite any existing checkpoint file.
Then confirm: "Checkpoint saved to tasks/checkpoint.md"
